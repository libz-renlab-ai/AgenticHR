你是招聘信息抽取助手。下面是 HR 和候选人的完整聊天对话，每条消息前有 [#编号]。

任务：对每个 pending 字段，找出**候选人**说过的、与该字段相关的消息编号。
不要改写、不要归一化、不要拆分、不要省略否定词，**只输出编号**。
后端会把你选的编号对应的整条消息原文拼起来作为 value——这样无论候选人
说得多碎，原话都会完整保留。

规则：
1. 只考虑标记为"候选人"的消息；HR 消息一律忽略。
2. 字段 → 输出**消息编号数组**（int 数组，按对话时间顺序），无相关消息则输出空数组 []。
3. 一条消息可同时归属多个字段（比如同一条消息既谈到了到岗时间又谈到了实习时长，必须 *三个* 数组都包含该编号）。
4. 否定语义按真实归属判断：候选人说"明天晚上没空"属于 free_slots（在表达面试时间），
   不属于 arrival_date。
5. **纯问候 / 纯打听岗位 / 纯表达兴趣** 的消息（如"还在招吗"、"我很感兴趣"、"希望进一步沟通"），
   不属于任何字段 → 所有字段都不要包含它的编号。**宁可漏报也不要硬塞**。

对话：
"""
{conversation}
"""

需要抽取的字段 pending_keys={pending_keys}：
- arrival_date: 候选人提到入职/到岗时间。关键词锚: "到岗 / 入职 / 立即 / 马上 / 随时 /
  下周X / 本周 / 明天 / 月 / 号 / 答辩后 / 毕业后"。
- intern_duration: 候选人提到实习能做多久。关键词锚: "实习 / X 个月 / 半年 / X 周 / 长期 /
  短期 / 每周 X 天 / 出勤 / 全职 / 兼职"。
- free_slots: 候选人提到面试可/不可时间。关键词锚: "面试 / 时段 / 上午 / 下午 / 晚上 /
  周一-周日 / X 点 / 有空 / 没空 / 约时间"。

正例 1（复合句 → 同一编号归属三个字段）：
  [#0] 候选人: 我目前可立即到岗，每周出勤五天，可实习6个月。
  → {{"arrival_date": [0], "intern_duration": [0], "free_slots": [0]}}
  （"立即到岗" → arrival_date；"实习6个月" → intern_duration；"每周出勤五天" → free_slots）

正例 2（分散消息 → 各取相应编号）：
  [#1] 候选人: 下周一可以到岗
  [#3] 候选人: 周三下午、周五上午有空
  → {{"arrival_date": [1], "intern_duration": [], "free_slots": [3]}}

反例（纯问候/打听 → 所有字段都为空）：
  [#0] 候选人: 您好，请问全栈工程师还在招吗？我很感兴趣，希望可以进一步沟通。
  → {{"arrival_date": [], "intern_duration": [], "free_slots": []}}
  （"希望可以"是礼貌话术不是面试时间；整条消息没有任何时间/时长/到岗信息）

只输出 JSON，无其他文字。
（null 与 [] 等价，都表示候选人没说该字段）

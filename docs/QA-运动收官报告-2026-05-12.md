## 测试基建关键决策

1. **独立 test DB**: `data/qa_test_<round>.db`,每轮起新副本
2. **session-level engine rebind**: `qa_engine_bound` autouse fixture 把 `app.database.engine` 重绑到 qa_test.db,解 in-process service 调用走错库的根因
3. **trust_env=False**: httpx + Playwright 双层绕本地代理拦截 127.0.0.1
4. **utf-8 subprocess**: claude CLI 输出 UTF-8,Windows 默认 GBK 解码挂线程,需显式 `encoding="utf-8"`
5. **vite via 直接 .cmd**: 绕 pnpm 包装路径空格 quoting 问题
6. **verifier 模型**: claude haiku-4-5(单次 ~$0.03,100 项 ~$3)
7. **预算守卫**: `BudgetGuard` 监控 ASR/LLM/会议/Boss 配额,超额抛 `BudgetExceeded`
